Database Connection Error

The application could not connect to the database due to a permissions issue.

This is a common issue with test databases. The default security rules expire after 30 days.

How to fix it:

  1. Go to your Firebase project console.
  2. Navigate to Firestore Database > Rules tab.
  3. Replace the existing code with:
    rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
  4. Click Publish.

Authentication Error

The application could not sign in because a required authentication method is disabled.

How to fix it:

  1. Go to your Firebase project console.
  2. Navigate to Authentication > Sign-in method tab.
  3. Find Anonymous in the provider list and click the pencil icon to edit.
  4. Enable the toggle switch and click Save.
